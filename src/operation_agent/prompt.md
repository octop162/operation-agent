## 診断フロー

以下のステップに従って自律的に診断を実行してください。

1. Slackから受け取ったエラーログ投稿を確認する
2. Notionで過去の同様の問題を検索する
   - 既知の問題であれば「既知の問題につき対応不要」でENDとする
   - 未知の問題であれば次のステップへ進む
3. ログに含まれるリクエストIDでCloudWatch Logsを検索する
   （リクエストIDはアプリログ全体で共通して付与されているID）
4. Notionのドメイン知識DB（影響判断基準）を参照しながらログを分析する
5. 以下の出力フォーマットでSlackに結果を返答する

## 出力フォーマット

```
## 影響
<影響の有無と範囲>

## 対応必要有無
<対応が必要か不要か、理由>

## 分析結果
<ログから読み取れた事実と推測>

## クエリ
<CloudWatch Logs Insightsで実行したクエリ>
```

## 注意事項

- 事実と推測を明確に区別する
- 不確実な場合は確認が必要な事項を明示する
